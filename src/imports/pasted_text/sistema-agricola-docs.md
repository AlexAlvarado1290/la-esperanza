Objetivo del sistema

La aplicación debe permitir:
	•	registro e inicio de sesión de usuarios
	•	gestión de productores y compradores por parte del administrador
	•	registro de productos agrícolas
	•	actualización de cantidades disponibles
	•	consulta de productos disponibles
	•	registro de solicitudes de compra
	•	registro y seguimiento de acuerdos comerciales
	•	programación de entregas
	•	registro de entregas realizadas
	•	control de pagos
	•	consulta de historial y reportes

Estilo visual

Crear una interfaz:
	•	limpia y moderna
	•	amigable y fácil de entender
	•	con botones grandes
	•	textos claros y legibles
	•	navegación sencilla
	•	pocos elementos por pantalla
	•	ideal para entorno rural y usuarios no técnicos

Paleta de colores sugerida
	•	verde agrícola como color principal
	•	blanco como fondo principal
	•	gris claro para tarjetas y secciones
	•	negro o gris oscuro para texto
	•	amarillo suave o naranja para alertas y estados pendientes
	•	rojo solo para errores o cancelaciones

Tipografía
	•	sans serif moderna y clara
	•	jerarquía visual marcada
	•	títulos grandes
	•	etiquetas simples
	•	formularios fáciles de leer

Pantallas que debe incluir
	1.	Pantalla de inicio de sesión

	•	logo o nombre del sistema
	•	campos de usuario y contraseña
	•	botón iniciar sesión
	•	mensaje de ayuda o acceso simple

	2.	Dashboard del administrador

	•	resumen general
	•	total de productores
	•	total de compradores
	•	productos registrados
	•	solicitudes pendientes
	•	entregas programadas
	•	pagos pendientes
	•	accesos rápidos a módulos principales

	3.	Pantalla de registro de productor

	•	formulario con datos personales
	•	nombre
	•	teléfono
	•	dirección o comunidad
	•	usuario
	•	contraseña
	•	botón guardar

	4.	Pantalla de registro de comprador

	•	formulario similar al de productor
	•	opción para tipo de comprador
	•	botón guardar

	5.	Pantalla de listado de usuarios

	•	tarjetas o tabla simple
	•	filtro por productor o comprador
	•	acción editar
	•	acción activar o desactivar

	6.	Pantalla de registro de producto

	•	nombre del producto
	•	categoría o tipo
	•	descripción
	•	unidad de medida
	•	precio referencial
	•	cantidad disponible
	•	botón guardar

	7.	Pantalla de productos disponibles

	•	listado en tarjetas
	•	nombre del producto
	•	productor
	•	cantidad disponible
	•	precio
	•	botón ver detalle
	•	botón solicitar compra

	8.	Pantalla de solicitud de compra

	•	selección de productos
	•	cantidad requerida
	•	observaciones
	•	botón enviar solicitud

	9.	Pantalla de acuerdos comerciales

	•	listado de acuerdos
	•	productor
	•	comprador
	•	productos
	•	precio pactado
	•	fecha de entrega
	•	estado del acuerdo
	•	botón ver detalle

	10.	Pantalla de detalle del acuerdo comercial

	•	información completa del acuerdo
	•	cantidades aceptadas
	•	precio final
	•	punto de entrega
	•	fecha
	•	estado
	•	botón actualizar estado
	•	botón programar entrega

	11.	Pantalla para programar entrega

	•	fecha de entrega
	•	hora
	•	punto de encuentro
	•	observaciones
	•	botón guardar

	12.	Pantalla de registro de entrega realizada

	•	acuerdo asociado
	•	cantidad entregada
	•	observaciones
	•	botón confirmar entrega

	13.	Pantalla de registro de pago

	•	acuerdo relacionado
	•	monto pagado
	•	fecha de pago
	•	estado del pago
	•	botón guardar

	14.	Pantalla de historial y reportes

	•	filtros por productor, comprador, producto, fecha o estado
	•	tarjetas o tabla con historial
	•	resumen visual simple
	•	reportes básicos

Navegación

La navegación debe ser simple y clara. Usar:
	•	barra lateral en escritorio
	•	barra inferior o menú hamburguesa en móvil
	•	iconos simples
	•	acceso rápido a módulos principales

Componentes UI

Usar:
	•	tarjetas limpias
	•	tablas simples
	•	formularios de un solo flujo
	•	botones grandes
	•	badges de estado como pendiente, confirmado, entregado, pagado, cancelado
	•	diseño responsive mobile-first

Estados del sistema

Incluir visualmente estados como:
	•	pendiente
	•	confirmado
	•	en proceso
	•	entregado
	•	cancelado
	•	pagado parcialmente
	•	pagado

Enfoque UX

La aplicación debe transmitir:
	•	facilidad de uso
	•	organización
	•	transparencia
	•	control de ventas y entregas
	•	confianza para usuarios rurales y administradores comunitarios
